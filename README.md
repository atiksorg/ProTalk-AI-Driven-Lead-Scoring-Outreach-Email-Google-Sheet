# ProTalk: AI-Driven Lead Scoring and Outreach Email in Google Sheet

**Комбайн v4.0** — автоматизация холодных email‑рассылок с глубоким AI‑анализом сайтов, скорингом лидов и персонализированной генерацией писем. Всё работает прямо в Google Sheets и использует функции платформы [ProTalk](https://pro-talk.ru).

---

## 🔧 Настройка ProTalk и используемые функции

Для работы Комбайна необходима регистрация на платформе [ProTalk](https://pro-talk.ru). После регистрации вы получите:

- `BOT_ID` и `BOT_TOKEN` — идентификаторы вашего бота, через которые выполняются все вызовы функций.
- Личный кабинет с балансом токенов (токены приобретаются внутри платформы).

**Комбайн использует следующие функции ProTalk:**

| ID функции | Название                                      | Стоимость (токенов ProTalk)        | Назначение в Комбайне                                      |
|------------|-----------------------------------------------|------------------------------------|------------------------------------------------------------|
| **489**    | Поиск в Google (SerpAPI)                      | 100 000 токенов за вызов           | Поиск сайтов по заданным запросам                          |
| **311**    | Получение скриншота сайта v2                  | 150 000 токенов за вызов           | Снятие скриншота и получение HTML для анализа              |
| **640**    | Омниканальный вопрос к ИИ (Vision)            | динамически (≈50 000 за анализ)    | Анализ скриншота по промпту (оценка дизайна, возраста и т.п.) |
| **57**     | Отправить уведомление в Telegram              | **бесплатно**                      | Оповещения о Hot‑Leads и важных событиях                   |

Убедитесь, что ваш баланс ProTalk позволяет выполнять запланированный объём работы. Пополнить баланс можно в личном кабинете ProTalk.

## ✨ Ключевые возможности

- 🔍 **Автоматический поиск сайтов** по поисковым запросам (Google через ProTalk Search)
- 🧠 **Глубокий анализ сайта** – извлечение CMS, контактов, адаптивности, аналитики, соцсетей и e‑commerce сигналов
- 🖼 **Vision‑анализ скриншотов** – оценка дизайна, возраста сайта, соответствия критериям (ProTalk Vision)
- 🎯 **Скоринг лидов** – автоматическая оценка привлекательности сайта по 10+ параметрам
- ✉️ **Генерация писем с помощью AI** – три режима работы ИИ (`strict`, `rewrite`, `full`) через AI Router
- 🔄 **Секвенции (follow‑up)** – автоматическая отправка серии писем с заданными задержками
- ⚖️ **A/B‑тестирование промптов** – автоматический выбор лучшего варианта ледокола
- 🔥 **Hot‑Leads с ручным или автоматическим утверждением**
- 🛡️ **Дедупликация доменов** – повторная отправка не ранее чем через 90 дней
- 📬 **Интеграция с Gmail** – отправка писем от имени владельца скрипта
- 📊 **Детальное логирование** – все действия записываются в лог‑лист
- 🔔 **Telegram‑уведомления** – оповещения о новых Hot‑Leads (бесплатная функция ProTalk)
- ⏱ **Расписание и лимиты** – работа только в заданные дни/часы, ограничение количества писем в сутки

## 📦 Требования

- Аккаунт Google (Google Sheets и Google Apps Script)
- Аккаунт и пополненный баланс в [ProTalk](https://pro-talk.ru)
- Токен OpenRouter (передаётся через AI Router ProTalk)
- (Опционально) Telegram‑бот для уведомлений

## 🚀 Установка и настройка

1. **Создайте копию Google‑таблицы**  
   Скачайте пустую таблицу‑шаблон или создайте новую и выполните скрипт `setupSheets()` (см. ниже).

2. **Откройте редактор скриптов**  
   В таблице выберите `Расширения` → `Apps Script`. Удалите содержимое по умолчанию.

3. **Загрузите файлы проекта**  
   Скопируйте содержимое каждого `.gs` файла из репозитория в соответствующие файлы скрипта:
   - `Code.gs`
   - `Campaigns.gs`
   - `HTMLParser.gs`
   - `ProTalkAPI.gs`
   - `Scoring.gs`
   - `Setup.gs`
   - `Utils.gs`

   *Названия файлов должны точно совпадать (регистр важен).*

4. **Настройте конфигурацию**  
   Запустите функцию `setupSheets()` из меню «Комбайн v4.0» → «⚙️ Создать/Обновить листы».  
   Будут созданы все необходимые листы с примерами данных.

5. **Заполните лист `⚙️ CONFIG`**  
   - `BOT_ID` и `BOT_TOKEN` – данные вашего бота ProTalk.
   - `AUTH_TOKEN` – Bearer‑токен для AI Router (OpenRouter).
   - `USER_EMAIL` – ваш email для идентификации в AI Router.
   - `FILE_UPLOAD_TOKEN` – токен для загрузки скриншотов в постоянное хранилище (опционально, но рекомендуется).
   - `SENDER_NAME` – имя отправителя писем.
   - `TG_BOT_TOKEN` и `TG_CHAT_ID` – для Telegram‑уведомлений (опционально).

6. **Настройте кампании, запросы и шаблоны**  
   - **`🎯 CAMPAIGNS`** – создайте активную кампанию, укажите режим (`full`/`html_only`/`skip`), пороги скоринга, промпты.
   - **`🔍 QUERIES`** – добавьте поисковые запросы для кампании.
   - **`🔄 SEQUENCES`** – опишите шаги рассылки (задержки в днях).
   - **`✉️ EMAIL_TEMPLATES`** – задайте тему, текст оффера, фоллоу‑ап, подпись и **режим работы ИИ** (`strict`, `rewrite`, `full`).

7. **Установите триггер**  
   В редакторе скриптов нажмите на иконку часов (Триггеры) и создайте триггер для функции `runCombine` с интервалом, например, каждые 30 минут.  
   *Или используйте пункт меню «⏱️ Создать триггер (30 мин)».*

## ⚙️ Как это работает

1. **Поиск новых сайтов**  
   При каждом запуске `runCombine()` скрипт проверяет дневной лимит и рабочее время. Если есть свободный «слот», выбирается случайная активная кампания и случайный поисковый запрос, затем через **ProTalk Search (функция 489)** получается список URL.

2. **Анализ сайта**  
   Для каждого URL (в режиме `full`) делается скриншот и извлекается HTML через **ProTalk Screenshot (функция 311)**.  
   - **HTMLParser** собирает CMS, контакты, адаптивность, аналитику, соцсети.  
   - **Vision API (функция 640)** при наличии скриншота оценивает сайт по заданному промпту.  
   - **Скоринг** вычисляет итоговый балл.

3. **Принятие решения**  
   - Если балл ниже порога – лид отклоняется.  
   - Если балл ≥ порога `hotLeadScore`, лид попадает в `🔥 HOT_LEADS`.  
     - При авто‑отправке письмо уходит сразу.  
     - Иначе ждёт ручного решения (`send` / `skip`).  
   - Если балл между порогами – отправляется первое письмо секвенции.

4. **Генерация письма**  
   Скрипт выбирает вариант промпта (A/B) и использует **AI Router** (через ProTalk) для создания ледокола или всего письма в соответствии с режимом из `EMAIL_TEMPLATES`:
   - **`strict`** – ИИ пишет только персональный ледокол, оффер вставляется без изменений.
   - **`rewrite`** – ИИ адаптирует оффер под сайт, сохраняя все ссылки и числа.
   - **`full`** – ИИ пишет письмо целиком, оффер служит лишь брифом.

5. **Отправка и логирование**  
   Письмо отправляется через GmailApp, threadId сохраняется. В лог записываются статус, скор, URL, email, скриншот, сгенерированное тело письма и ответ Vision.

6. **Follow‑up**  
   При следующих запусках скрипт проверяет задачи со статусом `wait_seq_N`. Если прошло достаточно дней и от клиента нет ответа, отправляется следующий шаг секвенции.

## 🕹 Использование

- **Запуск вручную**: откройте таблицу, в меню выберите `🤖 Комбайн v4.0` → `▶️ Запустить сейчас`.
- **Автоматический запуск**: после настройки триггера скрипт будет выполняться по расписанию.
- **Проверка соединения**: пункты `🔌 Тест ProTalk API` и `📨 Тест Telegram` помогут убедиться в корректности настроек.
- **Ручное управление Hot‑Leads**: в листе `🔥 HOT_LEADS` установите в колонке H значение `send` или `skip` – при следующем запуске решение будет обработано.

## 🔌 Интеграции

- **ProTalk API** – универсальный шлюз для выполнения функций:
  - Поиск в Google (функция 489)
  - Скриншот сайта + HTML (функция 311)
  - Vision‑анализ изображения (функция 640)
  - Telegram‑уведомления (функция 57, бесплатно)
- **AI Router (OpenRouter)** – вызов языковых моделей через ProTalk с поддержкой структурированного вывода.
- **Gmail API** – отправка писем, работа с тредами (используется встроенный сервис `GmailApp`).
- **Telegram Bot API** – уведомления о новых Hot‑Leads.
